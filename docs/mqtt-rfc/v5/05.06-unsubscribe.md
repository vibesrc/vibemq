## 5.6.1 UNSUBSCRIBE – Unsubscribe Request

### Fixed Header: 0xA2 (bits 3-0 MUST be 0010)

### Variable Header:
1. Packet Identifier (2 bytes)
2. Properties (User Property only)

### Payload:
List of Topic Filters (UTF-8 strings) to unsubscribe from.

**[MQTT-3.10.3-1]** The Payload MUST contain at least one Topic Filter.

---

## 5.6.2 UNSUBACK – Unsubscribe Acknowledgment

### Fixed Header: 0xB0

### Variable Header:
1. Packet Identifier (2 bytes)
2. Properties (Reason String, User Property)

### Payload:
List of Reason Codes, one per Topic Filter.

### UNSUBACK Reason Codes

| Value | Reason Code |
|-------|-------------|
| 0x00 | Success |
| 0x11 | No subscription existed |
| 0x80 | Unspecified error |
| 0x83 | Implementation specific error |
| 0x87 | Not authorized |
| 0x8F | Topic Filter invalid |
| 0x91 | Packet Identifier in use |

**[MQTT-3.10.4-1]** The Server MUST respond with UNSUBACK even if no Topic Filters matched.
