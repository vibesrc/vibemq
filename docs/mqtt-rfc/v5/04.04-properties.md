## 4.4.1 Overview

Properties are a new feature in MQTT 5.0. A Property consists of an Identifier which defines its usage and data type, followed by a value.

A Control Packet which contains an Identifier which is not valid for its packet type, or contains a value not of the specified data type, is a Malformed Packet.

## 4.4.2 Property Types

| Type | Description |
|------|-------------|
| Byte | Single unsigned byte (0-255) |
| Two Byte Integer | 16-bit unsigned integer, big-endian |
| Four Byte Integer | 32-bit unsigned integer, big-endian |
| Variable Byte Integer | 1-4 byte encoded integer |
| UTF-8 Encoded String | Length-prefixed UTF-8 string |
| UTF-8 String Pair | Two consecutive UTF-8 strings (name, value) |
| Binary Data | Length-prefixed binary data |

## 4.4.4 Property Details

### 4.4.4.1 Payload Format Indicator (0x01)

**Type:** Byte  
**Packets:** PUBLISH, Will Properties

- 0 = Unspecified bytes (default)
- 1 = UTF-8 encoded character data

If set to 1, the payload MUST be valid UTF-8. The Server MAY validate this and send PUBACK/PUBREC with Reason Code 0x99 (Payload format invalid) if validation fails.

### 4.4.4.2 Message Expiry Interval (0x02)

**Type:** Four Byte Integer  
**Packets:** PUBLISH, Will Properties

Lifetime of the message in seconds. If absent, the message does not expire.

When a Server publishes a message to a subscriber, it MUST set the Message Expiry Interval to the received value minus the time the message has been waiting in the Server.

### 4.4.4.3 Content Type (0x03)

**Type:** UTF-8 Encoded String  
**Packets:** PUBLISH, Will Properties

Describes the content of the Application Message. Typically a MIME type (e.g., "application/json").

### 4.4.4.4 Response Topic (0x08)

**Type:** UTF-8 Encoded String  
**Packets:** PUBLISH, Will Properties

Used for request/response patterns. Contains the Topic Name for the response message.

### 4.4.4.5 Correlation Data (0x09)

**Type:** Binary Data  
**Packets:** PUBLISH, Will Properties

Used by the sender of the request to identify which request a response corresponds to.

### 4.4.4.6 Subscription Identifier (0x0B)

**Type:** Variable Byte Integer  
**Packets:** PUBLISH, SUBSCRIBE

In SUBSCRIBE: Associates an identifier with the subscription.
In PUBLISH: Contains identifiers of matching subscriptions.

Value MUST be 1 to 268,435,455. Value 0 is a Protocol Error.

Multiple Subscription Identifiers can appear in a PUBLISH if the message matches multiple subscriptions.

### 4.4.4.7 Session Expiry Interval (0x11)

**Type:** Four Byte Integer  
**Packets:** CONNECT, CONNACK, DISCONNECT

Time in seconds after disconnection before the Session expires.

- 0 = Session ends when Network Connection closes
- 0xFFFFFFFF = Session does not expire

In DISCONNECT, can be used to change the Session Expiry Interval, but cannot be set to non-zero if it was zero in CONNECT.

### 4.4.4.8 Assigned Client Identifier (0x12)

**Type:** UTF-8 Encoded String  
**Packets:** CONNACK

Client Identifier assigned by the Server when the Client connected with a zero-length Client Identifier.

### 4.4.4.9 Server Keep Alive (0x13)

**Type:** Two Byte Integer  
**Packets:** CONNACK

Keep Alive time the Server requires. If present, the Client MUST use this value instead of the Keep Alive sent in CONNECT.

### 4.4.4.10 Authentication Method (0x15)

**Type:** UTF-8 Encoded String  
**Packets:** CONNECT, CONNACK, AUTH

Name of the authentication method (e.g., "SCRAM-SHA-256", "GS2-KRB5").

### 4.4.4.11 Authentication Data (0x16)

**Type:** Binary Data  
**Packets:** CONNECT, CONNACK, AUTH

Data for the authentication exchange.

### 4.4.4.12 Request Problem Information (0x17)

**Type:** Byte  
**Packets:** CONNECT

- 0 = Server MAY return Reason String or User Properties on failures only
- 1 = Server MAY return Reason String or User Properties on any packet

Default is 1.

### 4.4.4.13 Will Delay Interval (0x18)

**Type:** Four Byte Integer  
**Packets:** Will Properties

Delay in seconds before the Server publishes the Will Message after the Network Connection closes. Default is 0.

### 4.4.4.14 Request Response Information (0x19)

**Type:** Byte  
**Packets:** CONNECT

- 0 = Server MUST NOT return Response Information
- 1 = Server MAY return Response Information in CONNACK

### 4.4.4.15 Response Information (0x1A)

**Type:** UTF-8 Encoded String  
**Packets:** CONNACK

Used to construct Response Topics. Typically a topic prefix the Client is authorized to use.

### 4.4.4.16 Server Reference (0x1C)

**Type:** UTF-8 Encoded String  
**Packets:** CONNACK, DISCONNECT

Reference to another Server the Client should use. Format depends on the transport being used.

### 4.4.4.17 Reason String (0x1F)

**Type:** UTF-8 Encoded String  
**Packets:** CONNACK, PUBACK, PUBREC, PUBREL, PUBCOMP, SUBACK, UNSUBACK, DISCONNECT, AUTH

Human-readable string for diagnostics. Should not be parsed by the Client.

### 4.4.4.18 Receive Maximum (0x21)

**Type:** Two Byte Integer  
**Packets:** CONNECT, CONNACK

Maximum number of QoS 1 and QoS 2 PUBLISH packets that can be sent without receiving acknowledgment. Default is 65,535. Value of 0 is a Protocol Error.

### 4.4.4.19 Topic Alias Maximum (0x22)

**Type:** Two Byte Integer  
**Packets:** CONNECT, CONNACK

Maximum Topic Alias value the sender will accept. Default is 0 (no Topic Aliases accepted).

### 4.4.4.20 Topic Alias (0x23)

**Type:** Two Byte Integer  
**Packets:** PUBLISH

Value identifying the Topic. Value of 0 or greater than Topic Alias Maximum is a Protocol Error.

### 4.4.4.21 Maximum QoS (0x24)

**Type:** Byte  
**Packets:** CONNACK

Maximum QoS the Server supports (0, 1, or 2). If absent, Server supports QoS 2.

### 4.4.4.22 Retain Available (0x25)

**Type:** Byte  
**Packets:** CONNACK

- 0 = Retained messages not supported
- 1 = Retained messages supported (default)

### 4.4.4.23 User Property (0x26)

**Type:** UTF-8 String Pair  
**Packets:** All packets with Properties

Application-specific metadata. Can appear multiple times.

### 4.4.4.24 Maximum Packet Size (0x27)

**Type:** Four Byte Integer  
**Packets:** CONNECT, CONNACK

Maximum total packet size in bytes. If absent, no limit beyond protocol maximum.

### 4.4.4.25 Wildcard Subscription Available (0x28)

**Type:** Byte  
**Packets:** CONNACK

- 0 = Wildcard Subscriptions not supported
- 1 = Wildcard Subscriptions supported (default)

### 4.4.4.26 Subscription Identifier Available (0x29)

**Type:** Byte  
**Packets:** CONNACK

- 0 = Subscription Identifiers not supported
- 1 = Subscription Identifiers supported (default)

### 4.4.4.27 Shared Subscription Available (0x2A)

**Type:** Byte  
**Packets:** CONNACK

- 0 = Shared Subscriptions not supported
- 1 = Shared Subscriptions supported (default)
